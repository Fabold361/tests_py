# tests python

Kurze Zusammenfassung: Sammlung alter Klausuren (WS 2025) für Programmieren (Python) aus dem Elektrotechnik-Studium an der TU Wien.

Dieses Projekt ist mein persönliches Repository für Python-Übungen und alte Prüfungen. Ich sammle hier alle Laboraufgaben und Testdateien, damit ich später gezielt lernen und üben kann.

## Projektstruktur

- `lab1/` - Aufgaben und Testdateien aus Labor 1 (Python)
  - `Angabe/` - Angabe als PDF/Bild
  - `Code/` - Python Code
- `lab2/` - Aufgaben und Testdateien aus Labor 2 (Python)
  - `Angabe/` - Angabe als PDF/Bild
  - `Code/` - Python Code
- `lab3/` - Aufgaben und Testdateien aus Labor 3 (Python)
  - `Angabe/` - Angabe als PDF/Bild
  - `Code/` - Python Code
- `lab4/` - Aufgaben und Testdateien aus Labor 4 (Python)
  - `Angabe/` - Angabe als PDF/Bild
  - `Code/` - Python Code
- `lab5/` - Platzhalterordner für Labor 5 (Python)
- `lab6/` - Platzhalterordner für Labor 6 (Python)
- `.gitignore` - Ignoriert Python Cache und Build-Dateien
- `README.md` - Projektbeschreibung und Hinweise

## GitHub Repository

`github.com/Fabold361/tests_python`

## Nutzung

1. Repo klonen und im Root-Ordner (`tests_python/`) in VS Code öffnen.
2. In den jeweiligen `labX/Code/`-Ordner navigieren und die gewünschte Datei ausführen.
3. Die Dateien sind so organisiert, dass jede Laboraufgabe ihren eigenen Ordner bekommt.

## Ausführen

Vom Root-Ordner (`tests_python/`) aus:

```bash
python3 labX/Code/taskX.py
```

Beispiel für Lab 1:

```bash
python3 lab1/Code/task.py
```

## Hinweise

- Wenn neue Labore oder Prüfungsaufgaben hinzukommen, einfach einen neuen Ordner `labX/` mit `Angabe/` und `Code/` anlegen.
- Build-Dateien und Python Cache werden via `.gitignore` nicht ins Repo gepusht.
